<objective>
Create a device capability detection system that determines the optimal inference tier (cloud/hybrid/edge) based on device memory, WebGPU availability, and storage quota. This is the foundation for the hybrid inference architecture.
</objective>

## Task 1: Implement capability detection module
- **read_first**: `apps/frontend/src/` for existing state/storage patterns
- **action**: Create `apps/frontend/src/lib/device-capability.ts` with:
  - `navigator.deviceMemory` check (fallback to 2GB)
  - WebGPU adapter availability test via `navigator.gpu?.requestAdapter()`
  - Storage quota estimation via `navigator.storage.estimate()`
  - Feature detection for SharedArrayBuffer and WASM SIMD
- **acceptance_criteria**:
  - Module exports `detectDeviceCapabilities()` function
  - Returns typed `DeviceCapabilities` object with memory, webgpu, storage fields
  - Handles browsers without API support gracefully

## Task 2: Implement tier assignment logic
- **read_first**: Task 1 output
- **action**: Create `apps/frontend/src/lib/tier-assignment.ts` with:
  - Tier 1 (cloud): < 4GB RAM or no WebGPU
  - Tier 2 (hybrid): 4-8GB RAM with limited WebGPU
  - Tier 3 (edge): 8GB+ RAM with full WebGPU support
  - Include tier assignment reasoning in return value
- **acceptance_criteria**:
  - Function `selectInferenceTier()` returns `'cloud' | 'hybrid' | 'edge'`
  - Tier assignment is deterministic and capability-driven
  - Includes `reason` field explaining the assignment

## Task 3: Add tier persistence and diagnostics
- **read_first**: `apps/frontend/src/lib/tier-assignment.ts`
- **action**: Create `apps/frontend/src/lib/tier-storage.ts` with:
  - Persist tier decision to localStorage
  - Log tier assignment with device fingerprint
  - Track fallback events for diagnostics
- **acceptance_criteria**:
  - Tier decision persisted with timestamp
  - Diagnostic logs include device capabilities summary
  - Provides `getTierDiagnostics()` for debugging

## Task 4: Create React hook for device tier
- **read_first**: `apps/frontend/src/lib/` modules from Tasks 1-3
- **action**: Create `apps/frontend/src/hooks/useDeviceTier.ts` with:
  - Async tier detection on mount
  - Loading state during capability detection
  - Re-detection on visibility change (tab switch)
- **acceptance_criteria**:
  - Hook returns `{ tier, capabilities, isLoading, reason }`
  - Works with React 18+ strict mode
  - Memoizes expensive detection calls

## Task 5: Add unit tests for tier detection
- **read_first**: All previous task outputs
- **action**: Create `apps/frontend/src/lib/__tests__/device-capability.test.ts` with:
  - Mock navigator APIs for different device scenarios
  - Test tier assignment for low/mid/high-end devices
  - Test fallback behavior for missing APIs
- **acceptance_criteria**:
  - Tests cover all three tier assignments
  - Tests verify graceful degradation
  - All tests pass

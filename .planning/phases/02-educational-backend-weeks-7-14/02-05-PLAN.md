<objective>
Build the Google Drive sync engine that synchronizes IndexedDB learning state with user's Drive. Implements drive.appdata scope, conflict resolution (LWW baseline), compression, and periodic background sync.
</objective>

## Task 1: Configure Google Drive API integration
- **read_first**: `backend/auth/` for existing OAuth patterns
- **action**: Create `apps/frontend/src/lib/drive/auth.ts` with:
  - `drive.appdata` scope configuration
  - Token lifecycle management
  - Token refresh handling
  - Permission request flow
- **acceptance_criteria**:
  - Requests minimal `drive.appdata` scope
  - Handles token refresh automatically
  - Stores tokens securely
  - Provides auth status to sync engine

## Task 2: Create Drive API client
- **read_first**: Task 1 output
- **action**: Create `apps/frontend/src/lib/drive/api-client.ts` with:
  - File CRUD operations in appdata folder
  - Upload with resume support
  - Download with progress callback
  - Error handling and retry logic
- **acceptance_criteria**:
  - Implements Google Drive REST API v3
  - Handles rate limiting (exponential backoff)
  - Supports large file uploads (>5MB)
  - Provides upload/download progress

## Task 3: Implement sync engine
- **read_first**: `apps/frontend/src/lib/mastery/storage.ts`, Tasks 1-2 outputs
- **action**: Create `apps/frontend/src/lib/drive/sync-engine.ts` with:
  - IndexedDB → Drive sync flow
  - Drive → IndexedDB restore flow
  - Last-Write-Wins (LWW) conflict resolution
  - Sync status tracking
- **acceptance_criteria**:
  - Syncs mastery data to Drive
  - Restores from Drive on new device
  - Handles conflicts with LWW
  - Logs sync operations for debugging

## Task 4: Add compression for sync data
- **read_first**: `apps/frontend/src/lib/drive/sync-engine.ts`
- **action**: Create `apps/frontend/src/lib/drive/compression.ts` with:
  - LZ-based compression for JSON data
  - Target 10x compression ratio
  - Streaming compression for large datasets
  - Decompression with integrity check
- **acceptance_criteria**:
  - Compresses mastery data efficiently
  - Achieves 10x compression on typical data
  - Handles decompression errors
  - Maintains data integrity

## Task 5: Implement background sync scheduler
- **read_first**: `apps/frontend/src/lib/drive/sync-engine.ts`
- **action**: Create `apps/frontend/src/lib/drive/sync-scheduler.ts` with:
  - Periodic sync (15-30 minute intervals)
  - Background sync API integration
  - Network status detection
  - Battery-aware scheduling
- **acceptance_criteria**:
  - Syncs every 15-30 minutes when online
  - Pauses sync on cellular (configurable)
  - Respects battery saver mode
  - Syncs immediately after offline period

## Task 6: Create sync status UI integration
- **read_first**: All previous task outputs
- **action**: Create `apps/frontend/src/hooks/useSyncStatus.ts` with:

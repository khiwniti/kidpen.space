<objective>
Build the hybrid inference system that routes LLM requests to edge (WebLLM), hybrid, or cloud backends based on device tier. Includes model initialization, progressive download, and seamless failover handling.
</objective>

## Task 1: Create inference interface abstraction
- **read_first**: `backend/core/ai_models/` for existing model abstractions
- **action**: Create `apps/frontend/src/lib/inference/types.ts` with:
  - `InferenceBackend` interface with `generate()`, `isReady()`, `abort()` methods
  - `InferenceConfig` type for model parameters
  - `InferenceResult` type for response streaming
- **acceptance_criteria**:
  - Interface supports streaming responses
  - Interface supports abort/cancellation
  - Types are shared between edge and cloud implementations

## Task 2: Implement WebLLM edge backend
- **read_first**: `apps/frontend/src/lib/inference/types.ts`
- **action**: Create `apps/frontend/src/lib/inference/edge-backend.ts` with:
  - WebLLM engine initialization
  - Model loading with progress callback
  - Qwen2-0.5B-INT4 model configuration (400MB)
  - Phi-3-mini-INT4 model configuration (2.2GB) for Tier 3
  - OOM error detection and recovery
- **acceptance_criteria**:
  - Class implements `InferenceBackend` interface
  - Reports loading progress (0-100%)
  - Handles OOM by emitting fallback event
  - Supports model hot-swapping

## Task 3: Implement cloud fallback backend
- **read_first**: `backend/core/ai_models/providers/` for existing cloud patterns
- **action**: Create `apps/frontend/src/lib/inference/cloud-backend.ts` with:
  - HTTP client to backend inference API
  - Streaming response handling via SSE
  - Request timeout and retry logic
  - Authentication token injection
- **acceptance_criteria**:
  - Class implements `InferenceBackend` interface
  - Streams tokens from cloud API
  - Handles network errors gracefully
  - Supports request cancellation

## Task 4: Create inference router
- **read_first**: `apps/frontend/src/lib/tier-assignment.ts`, Tasks 1-3 outputs
- **action**: Create `apps/frontend/src/lib/inference/router.ts` with:
  - Route requests based on device tier
  - Automatic fallback from edge to cloud on error
  - Request queue management
  - Performance metrics collection
- **acceptance_criteria**:
  - Routes to correct backend based on tier
  - Falls back to cloud on edge failure
  - Logs routing decisions for diagnostics
  - Tracks latency per backend

## Task 5: Add progressive model download
- **read_first**: `apps/frontend/src/lib/inference/edge-backend.ts`
- **action**: Create `apps/frontend/src/lib/inference/model-downloader.ts` with:
  - Cache-aware model download (IndexedDB)
  - WiFi detection before large downloads
  - Download progress UI integration
  - Resume interrupted downloads
- **acceptance_criteria**:
  - Checks for existing cached models
  - Prompts user on cellular connection
  - Reports download progress percentage
  - Handles download interruption

## Task 6: Create React hook for inference
- **read_first**: All previous task outputs
- **action**: Create `apps/frontend/src/hooks/useInference.ts` with:
  - Unified inference API for components
  - Automatic backend selection
  - Loading/error states
  - Abort controller management
- **acceptance_criteria**:
  - Hook returns `{ generate, isLoading, error, abort }`
  - Works with streaming responses
  - Cleans up resources on unmount

## Task 7: Add integration tests
- **read_first**: All previous task outputs
- **action**: Create `apps/frontend/src/lib/inference/__tests__/` with:
  - Mock WebLLM engine tests
  - Cloud backend API mock tests
  - Router fallback scenario tests
  - End-to-end inference flow tests
- **acceptance_criteria**:
  - Tests cover all three backend types
  - Tests verify fallback behavior
  - Tests verify error handling
  - All tests pass

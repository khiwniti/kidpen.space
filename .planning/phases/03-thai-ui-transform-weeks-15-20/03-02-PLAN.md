<objective>
Transform the web application into a Progressive Web App (PWA) that works offline. Students should be able to access previously loaded content, view their mastery status, and queue learning actions for sync when connectivity returns.
</objective>

## Task 1: Service Worker Configuration
- **read_first**: Next.js PWA documentation
- **action**: Implement Service Worker using `next-pwa` or custom `sw.js` to cache static assets and the app shell.
- **acceptance_criteria**:
  - App shell loads without network connection.
  - Assets are served from cache first (Stale-while-revalidate).

## Task 2: Offline Asset Caching Strategy
- **read_first**: `apps/frontend/public/`
- **action**: Define caching rules for fonts, images, and core JS bundles.
- **acceptance_criteria**:
  - Sarabun font is cached.
  - Core UI icons and images are cached.

## Task 3: Offline UI & Feedback
- **read_first**: `apps/frontend/src/components/`
- **action**: Create `OfflineIndicator` component and integrate into the main layout.
- **acceptance_criteria**:
  - Visual indicator appears when network is lost.
  - "Offline" mode is clearly communicated to the student.

## Task 4: Background Sync Queue Integration
- **read_first**: `apps/frontend/src/lib/drive/sync-engine.ts`
- **action**: Use the `SyncEngine` to queue local changes while offline.
- **acceptance_criteria**:
  - Mastery updates are saved locally when offline.
  - Sync automatically resumes when connection is restored.

## Verification Steps

1. Enable "Offline" mode in browser dev tools.
2. Refresh the page and verify the app shell loads.
3. Perform a learning action (e.g., complete a quiz) and verify it's queued.
4. Restore connection and verify data syncs to Drive.

## Must-Haves

- [ ] App shell loads offline
- [ ] Offline indicator in UI
- [ ] Data integrity preserved during offline/online transition
- [ ] PWA "Install" prompt functional

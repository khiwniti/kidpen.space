<objective>
Ensure the platform provides a premium experience on low-end mobile devices common in rural Thailand. This includes touch-first navigation, 320px viewport support, and aggressive performance optimization.
</objective>

## Task 1: Responsive UI Audit (320px - 1920px)
- **read_first**: Existing Tailwind CSS configurations
- **action**: Refine responsive breakpoints and ensure all components scale down to 320px without overflow.
- **acceptance_criteria**:
  - No horizontal scrolling on 320px width.
  - Text is legible on small screens.

## Task 2: Touch-First Interaction Polish
- **read_first**: `Radix UI` accessibility guidelines
- **action**: Increase tap targets to at least 44x44px. Add swipe gestures for mobile navigation if appropriate.
- **acceptance_criteria**:
  - All buttons and links are easily tappable.
  - Dropdowns and modals work correctly on mobile browsers.

## Task 3: Performance Optimization for Slow Networks
- **read_first**: Web Vitals targets
- **action**: Implement image lazy loading, WebP conversion, and code splitting for large modules.
- **acceptance_criteria**:
  - LCP < 2.5s on 3G network simulation.
  - Total bundle size reduced by 20%.

## Verification Steps

1. Test the app on a physical 320px mobile device or simulator.
2. Verify touch interactions are smooth and responsive.
3. Check PageSpeed Insights for mobile performance score.
